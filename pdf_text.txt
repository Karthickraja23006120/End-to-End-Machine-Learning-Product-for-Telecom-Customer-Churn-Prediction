Domain 
 
 
Problem Statement: 
You are working as a Data Scientist for a telecom company. 
The company is facing high customer churn, which negatively impacts revenue and growth. 
You are given historical customer data containing: 
• 
Demographics 
• 
Account details 
• 
Services subscribed 
• 
Billing information 
Your task is to build an end-to-end Machine Learning product that: 
• 
Predicts whether a customer will churn 
• 
Explains why the customer is likely to churn 
• 
Helps the business take proactive retention actions 
• 
Exposes predictions via a production-ready API 
 
 
 
 
 
Business Use Cases 
• 
Identify customers at high risk of churn 
• 
Reduce customer acquisition cost 
• 
Improve customer lifetime value (CLV) 
• 
Enable personalized retention offers 
• 
Support management decisions with data-driven insights

3.2 Exploratory Data Analysis (EDA) 
Perform detailed analysis to answer key business questions: 
• 
Do senior citizens churn more? 
• 
How does tenure affect churn? 
• 
Which contract types have the highest churn? 
• 
Does monthly billing amount influence churn? 
• 
Which services are associated with higher churn?

• 
Churn distribution (class imbalance) 
• 
Tenure vs churn 
• 
MonthlyCharges vs churn 
•

• 
InternetService vs churn 
• 
Correlation heatmap 
 
3.3 Data Cleaning & Preprocessing 
• 
Convert TotalCharges to numeric (handle missing values) 
• 
Drop non-informative columns (customerID) 
• 
Handle missing and inconsistent values 
• 
Encode categorical variables 
• 
Scale numerical features (tenure, MonthlyCharges, TotalCharges) 
• 
Split data into Train / Validation / Test sets 
 
3.4 Feature Engineering 
Create meaningful features such as: 
• 
AvgMonthlySpend = TotalCharges / tenure 
• 
Tenure buckets (New / Medium / Loyal) 
• 
Service count per customer 
• 
Contract duration flags 
• 
High-value customer indicator 
Important: Each engineered feature must have business meaning. 
 
 
 
3.5 Handling Class Imbalance 
Since churn is imbalanced: 
• 
Use class weights or resampling techniques 
• 
Avoid accuracy-only evaluation 
• 
Focus on Recall, Precision, F1-score, ROC-AUC 
 
3.6 Model Building & Evaluation 
Train and compare multiple models: 
• 
Logistic Regression (baseline) 
• 
Random Forest 
• 
Gradient Boosting / XGBoost (Optional) 
Evaluation Metrics 
• 
ROC-AUC 
• 
Precision 
• 
Recall 
• 
F1-score 
• 
Confusion Matrix 
 
3.7 Threshold Optimization 
• 
Default threshold (0.5) is not mandatory 
• 
Choose optimal threshold based on: 
o Cost of losing a customer 
o Cost of retention incentives 
 
 
 
 
3.8 Model Explainability 
Apply Explainable AI techniques: 
• 
SHAP global feature importance 
• 
SHAP local explanations per customer 
Explain: 
• 
Why customers churn 
• 
Which features drive churn the most

Build a REST API using Flask or FastAPI.
